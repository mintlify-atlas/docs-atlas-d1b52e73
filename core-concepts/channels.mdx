## What are Channels?

Channels are the different communication mediums through which SuprSend can deliver notifications to your users. Each channel has its own characteristics, use cases, and implementation details.

<Tip>
Choose channels based on urgency, user preference, and message type. Critical alerts might use SMS, while newsletters work best via email.
</Tip>

## Channel Identifiers

In SuprSend, each channel is identified by a specific key:

| Channel | Key | Format |
|---------|-----|--------|
| Email | `$email` | Email addresses |
| SMS | `$sms` | Phone numbers with country code |
| WhatsApp | `$whatsapp` | Phone numbers with country code |
| Android Push | `$androidpush` | FCM/Xiaomi/OPPO tokens |
| iOS Push | `$iospush` | APNS tokens |
| Web Push | `$webpush` | VAPID subscription objects |
| Slack | `$slack` | Slack configuration objects |
| MS Teams | `$ms_teams` | Teams configuration objects |

## Email Channel

### Overview

Email is the most versatile channel, supporting rich HTML content, images, attachments, and complex layouts.

### Adding Email to User Profile

```python
user = supr_client.user.get_instance("user-123")
user.add_email("user@example.com")
user.save()
```

### Using Email in Workflows

```python
# Inline channel specification
wf = WorkflowTriggerRequest(
    body={
        "workflow": "welcome-email",
        "recipients": [
            {
                "distinct_id": "user-123",
                "$email": ["user@example.com", "backup@example.com"]
            }
        ],
        "data": {...}
    }
)
```

### Email Best Practices

- Use clear, actionable subject lines
- Include plain text alternatives
- Keep HTML emails under 100KB for best deliverability
- Test across multiple email clients
- Include unsubscribe links

## SMS Channel

### Overview

SMS provides immediate, text-only notifications with high open rates (98%+). Ideal for urgent, time-sensitive messages.

### Adding SMS to User Profile

```python
user = supr_client.user.get_instance("user-123")
# Always include country code
user.add_sms("+1234567890")  # US number
user.add_sms("+919876543210")  # India number
user.save()
```

<Warning>
Always include the country code (e.g., +1, +44, +91) when adding phone numbers. SMS delivery will fail without proper formatting.
</Warning>

### SMS Best Practices

- Keep messages under 160 characters to avoid splitting
- Include sender identification
- Use for time-sensitive information only
- Respect user preferences and quiet hours
- Be aware of SMS costs per message

## WhatsApp Channel

### Overview

WhatsApp enables rich, conversational notifications with support for images, buttons, and interactive elements.

### Adding WhatsApp to User Profile

```python
user = supr_client.user.get_instance("user-123")
user.add_whatsapp("+1234567890")  # Include country code

```

### WhatsApp Features

- Rich media support (images, videos, documents)
- Interactive buttons and quick replies
- Higher engagement than traditional SMS
- Template-based messaging for business use
- Read receipts and delivery status

### WhatsApp Best Practices

- Use approved message templates
- Include opt-in mechanisms
- Respond to user messages within 24 hours
- Balance promotional and transactional content

## Push Notifications

### Android Push

Supports multiple providers: FCM (Firebase Cloud Messaging), Xiaomi, and OPPO.

```python
user = supr_client.user.get_instance("user-123")

# FCM (default)
user.add_androidpush("fcm_token_here")

# Xiaomi
user.add_androidpush("xiaomi_token_here", provider="xiaomi")

# OPPO
user.add_androidpush("oppo_token_here", provider="oppo")

user.save()
```

### iOS Push

Uses Apple Push Notification service (APNS).

```python
user = supr_client.user.get_instance("user-123")
user.add_iospush("apns_token_here")
user.save()
```

### Push Token Management

<Info>
Push tokens can expire or change when users reinstall apps or update devices. Implement token refresh logic in your application.
</Info>

### Push Notification Best Practices

- Request notification permissions at the right time
- Use rich notifications (images, actions)
- Implement deep linking for better UX
- Handle token expiration and updates
- Respect notification preferences
- Test on both iOS and Android devices

## Web Push

### Overview

Web push enables browser notifications even when your website isn't open, using the VAPID protocol.

### Adding Web Push to User Profile

```python
user = supr_client.user.get_instance("user-123")

# Web push subscription object from browser
subscription = {
    "endpoint": "https://fcm.googleapis.com/fcm/send/...",
    "keys": {
        "p256dh": "key_here...",
        "auth": "auth_here..."
    }
}

user.add_webpush(subscription, provider="vapid")
user.save()
```

### Web Push Features

- Works across Chrome, Firefox, Safari, Edge
- Requires HTTPS
- Supports rich notifications with images and actions
- No app installation required
- User must grant permission

### Web Push Best Practices

- Ask for permission at appropriate times
- Explain the value before requesting permission
- Use notification icons and badges
- Implement notification click handlers
- Handle permission denials gracefully

## Slack Channel

### Overview

Integrate notifications into Slack workspaces via direct messages, channel posts, or incoming webhooks.

### Configuration Options

#### 1. Direct Message via Email

```python
user.add_slack({
    "email": "user@example.com",
    "access_token": "xoxb-your-bot-token"
})
```

#### 2. Direct Message via User ID

```python
user.add_slack({
    "user_id": "U03XXXXXXXX",
    "access_token": "xoxb-your-bot-token"
})
```

#### 3. Post to Channel

```python
user.add_slack({
    "channel_id": "C03XXXXXXXX",
    "access_token": "xoxb-your-bot-token"
})
```

#### 4. Incoming Webhook

```python
user.add_slack({
    "incoming_webhook": {
        "url": "https://hooks.slack.com/services/T00/B00/XXXX"
    }
})
```

### Slack Best Practices

- Use interactive components (buttons, dropdowns)
- Format messages with Block Kit
- Include relevant context and links
- Respect channel notification settings
- Test in a sandbox workspace first

## Channel Selection Strategies

### 1. Workflow-Level Channel Control

Specify channels for individual workflow triggers:

```python
wf = WorkflowTriggerRequest(
    body={
        "workflow": "payment-reminder",
        "recipients": [
            {
                "distinct_id": "user-123",
                "$channels": ["email", "sms"]  # Only use these channels
            }
        ]
    }
)
```

### 2. User Preferences

Respect user notification preferences stored in their profile:

```python
# Set user channel preferences
user.set("notification_preferences", {
    "email": True,
    "sms": False,
    "push": True
})
```

### 3. Template Configuration

Configure default channels in the SuprSend dashboard when creating workflow templates.

## Multi-Channel Strategies

### Redundancy Strategy

Send to multiple channels for critical messages:

```python
# Critical: Use email + SMS + push
wf = WorkflowTriggerRequest(
    body={
        "workflow": "security-alert",
        "recipients": [{
            "distinct_id": "user-123",
            "$channels": ["email", "sms", "push"]
        }]
    }
)
```

### Fallback Strategy

Automatic fallback if primary channel fails:

```python
# Try push first, fallback to email, then SMS
# Configure in workflow template on SuprSend dashboard
```

### Progressive Strategy

Escalate through channels based on user response:

```python
# 1. Send push notification
# 2. If not read in 1 hour, send email
# 3. If still not read in 4 hours, send SMS
# Configure using workflow delays in SuprSend dashboard
```

## Managing Multiple Channels

### Adding Multiple Channels

```python
user = supr_client.user.get_instance("user-123")

# Add multiple channel types
user.add_email("user@example.com")
user.add_sms("+1234567890")
user.add_whatsapp("+1234567890")
user.add_androidpush("fcm_token")
user.add_iospush("apns_token")

# Add multiple addresses for same channel
user.add_email("user@example.com")
user.add_email("backup@example.com")

user.save()
```

### Removing Channels

```python
# Remove specific channel value
user.remove_email("old@example.com")
user.remove_sms("+9876543210")

# Remove all values for a channel
user.unset("$email")
user.unset(["$sms", "$whatsapp"])

user.save()
```

## Compliance and Regulations

<Warning>
Always ensure compliance with:
- **CAN-SPAM Act** (Email)
- **TCPA** (SMS, Voice)
- **GDPR** (EU users, all channels)
- **CCPA** (California users, all channels)
- **Platform policies** (Apple, Google, WhatsApp, etc.)
</Warning>

### Best Practices for Compliance

1. **Obtain Consent**: Get explicit opt-in for marketing messages
2. **Provide Opt-Out**: Include unsubscribe mechanisms
3. **Respect Preferences**: Honor user channel preferences
4. **Data Privacy**: Protect user contact information
5. **Rate Limiting**: Avoid excessive notifications

## Channel Performance Monitoring

### Key Metrics to Track

- **Delivery Rate**: Percentage of messages successfully delivered
- **Open Rate**: Percentage of delivered messages opened
- **Click-Through Rate**: Percentage of users who clicked links
- **Conversion Rate**: Percentage who completed desired action
- **Unsubscribe Rate**: Users opting out per channel

### Optimizing Channel Performance

```python
# Track channel performance in events
Event(
    distinct_id="user-123",
    event_name="notification_sent",
    properties={
        "channel": "email",
        "workflow": "welcome-series",
        "message_id": "msg-123"
    }
)

Event(
    distinct_id="user-123",
    event_name="notification_opened",
    properties={
        "channel": "email",
        "message_id": "msg-123",
        "opened_at": "2024-02-28T10:30:00Z"
    }
)
```
